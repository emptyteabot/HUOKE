## 版本1: 完整版(推荐)

```
你好!我需要你作为我的全职技术合伙人,帮我完成一个AI驱动的留学获客系统。

## 项目信息

**GitHub仓库:** https://github.com/emptyteabot/HUOKE

**项目简介:**
这是一个留学行业的自动化获客系统,核心功能是:
1. 从社交平台(小红书/知乎/豆瓣/微博)自动搜索潜在客户
2. 提取客户联系方式和需求信息
3. 使用DeepSeek AI生成个性化邮件
4. 自动发送和追踪邮件效果
5. 基于行为信号评估客户意向

**当前状态:**
- ✅ 邮件营销系统已完成(DeepSeek集成)
- ✅ 信号驱动系统已完成
- ✅ AI评分系统已完成
- ❌ 最大问题:缺少真实客户数据(现有的是AI生成的假数据)
- ❌ 爬虫虽然能用,但抓不到联系方式

**技术栈:**
- Python 3.9+
- nodriver (异步爬虫)
- DeepSeek API (文案生成)
- SendGrid (邮件发送)

## 你的任务

请按以下顺序执行:

### 第1步: 理解项目(10分钟)

请访问GitHub仓库,阅读以下文件:
1. **README.md** - 了解项目概况
2. **GEMINI_CONTEXT.md** - 了解当前状态和核心问题
3. **Gemini获客Prompt工程.md** - 数据收集方法

然后回答:
- 这个项目要解决什么问题?
- 核心功能模块有哪些?
- 当前最大的痛点是什么?
- 技术架构是怎样的?

### 第2步: 收集真实客户数据(最优先!)

这是整个项目最关键的一步。请使用你的Deep Research功能:

**目标:** 收集150-200个真实的留学潜在客户

**数据来源:**
- 小红书 (xiaohongshu.com) - 50个
- 知乎 (zhihu.com) - 30个
- 豆瓣小组 (douban.com) - 20个
- 微博 (weibo.com) - 25个
- 留学论坛(寄托天下、一亩三分地) - 40个

**搜索关键词:**
- "留学中介推荐"
- "英国留学申请"
- "美国研究生DIY"
- "留学机构避坑"
- "求推荐靠谱中介"
- "留学申请求助"

**对每个客户提取:**

基本信息:
- 用户名/昵称
- 平台
- 用户主页链接

联系方式(如果公开):
- 邮箱
- 微信号
- QQ号
- 手机号

背景信息(从发帖内容推断):
- 本科学校
- 专业
- GPA/成绩
- 年级

目标信息:
- 目标国家
- 目标学校
- 目标专业
- 学位类型

意向分析:
- 预算范围
- 申请时间线
- 痛点(2-3个)
- 行为信号(搜索/咨询/对比等)

评分:
- 意向评分(1-10分)
- 优先级(S/A/B级)

**输出格式:**

以JSON格式输出,每个客户包含:

```json
{
  "name": "张同学",
  "platform": "小红书",
  "profile_url": "https://...",
  "contact": {

    "phone": "13812345678"
  },
  "background": {
    "school": "某985大学",
    "major": "计算机科学",
    "gpa": "3.5"
  },
  "target": {
    "country": "英国",
    "university": "UCL",
    "major": "CS"
  },
  "budget": "30-50万",
  "pain_points": ["不知道如何选校", "文书不会写"],
  "signals": ["在小红书搜索了UCL申请", "评论区咨询中介"],
  "intent_score": 8,
  "priority": "S",
  "collected_at": "2024-02-22 10:30:00"

```

### 第3步: 生成数据质量报告

## 使用建议

### 推荐顺序:

1. **第一次使用Gemini** → 用版本3(分步版)
   - 让Gemini先理解项目
   - 然后再逐步执行任务

2. **Gemini已经理解项目** → 用版本2(精简版)
   - 直接开始收集数据
   - 节省时间

3. **需要完整指导** → 用版本1(完整版)
   - 包含所有细节
   - 适合复杂任务

4. **只需要数据收集** → 用版本4(数据专用版)
   - 聚焦数据收集
   - 最快速度

### 使用技巧:

1. **复制粘贴** - 直接复制上面的Prompt,粘贴到Gemini
2. **等待回复** - Gemini会先分析项目,然后开始执行
3. **持续对话** - 如果数据不够,继续要求Gemini收集更多
4. **保存结果** - 把Gemini返回的JSON数据保存为 gemini_leads.json

### 后续对话:

**如果数据质量不好:**
```
数据质量不够好,请优化:
1. 联系方式完整率太低,请尽可能提取邮箱/微信
2. 意向评分不准确,请重新评估
3. 有些数据重复,请去重
```

**如果需要更多数据:**
```
很好!请继续收集:
- 再收集50个小红书客户
- 再收集30个知乎客户
保持相同的数据格式
```

**如果需要验证数据:**
```
请验证数据质量:
1. 检查邮箱格式是否正确
2. 检查是否有重复数据
3. 检查意向评分是否合理
4. 生成质量报告
```
